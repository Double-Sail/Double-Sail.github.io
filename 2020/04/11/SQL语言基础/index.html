2. 多表关联查询

2.1 表的别名

多表关联查询，多个表存在相同的列，必须使用表名来限定列的引用。但是查询变的复杂时，每次都需要限定就会显得十分冗长，所以需要设定表的别名。

举例：

SQL> select e.job 岗位,e.empno 员工编号,e.ename 员工名称,d.dname 部门 from emp e,dept d where e.deptno=d.deptno and e.job='MANAGER';

岗位       员工编号 员工名称   部门
--------- ----- ---------- --------------
MANAGER    7782 CLARK      ACCOUNTING
MANAGER    7566 JONES      RESEARCH
MANAGER    7698 BLAKE      SALES

注意：SQL不区分大小写，但是表中的内容区分大小写

-- manager用小写就无法显示。
SQL> select e.job 岗位,e.empno 员工编号,e.ename 员工名称,d.dname 部门 from emp e,dept d where e.deptno=d.deptno and e.job='manager';

岗位       员工编号 员工名称   部门
--------- ----- ---------- --------------

心得：表的关联相关SQL语句书写时，不是从前向后的写，而是先写好from后面的内容，再定义where子句（运 用已经定义过的表的别名，而不是原先的表名），最后写select后面的内容。

2.2 内连接

简单的说就是用inner join（inner可以省略）指定用于连接的两个表，使用on指定连接表的连接条件。进一步限定查询范围，使用where语句即可。

举例：

--通过deptno内连接emp和dept两张表，并检索两个表中相关字段的信息
SQL> select e.empno 员工编号，e.ename 员工名称，d.dname 部门 from emp e join dept d on e.deptno=d.deptno;

2.3 外连接

内连接只会显示on之后限定条件的行。而外连接可以显示不能满足限定条件的行

外连接主要分为左外连接，右外连接，完全外连接。

左、右外连接：即在查询结果中不仅包含了满足连接条件的数据行，还包含左表中不满足条件的行。右外连接的道 理类似。

举例：

SQL> insert into emp(empno,ename,job) values(9527,'YANGFAN','probie');
1 row inserted

SQL> select e.empno,e.ename,e.job,d.deptno,d.dname from emp e left join dept d on e.deptno=d.deptno;

​ 可以看到虽然插入数据行的deptno列值为NULL，但该行记录仍然出现在查询结果中，这说明左外连接的查询结果会包含左表中不满足“连接条件”的数据行。（右链接的道理类似）

完全外连接：当执行完全外连接时，Oracle会执行一个完整的左右外连接查询，然后将查询结果合并。并消除重 复的记录行。

2.4 自然连接

和内连接功能类似，指在检索多个表时，Oracle会将第一个中的列与第二个表中具有相同名称的列自动连接。自然连接中不需要明确指定进行连接的列，使用natural join关键字。

SQL> select empno,ename,job,dname from emp natural join dept where sal>2000;

2.5 自链接

主要运用在自参照表上显示上下级关系或者层次关系。自参照表即是在同一张表的不同列之间有参照关系或者主从关系。（自己和自己连接）

举例：查询所有管理者的下属员工信息。

SQL> select em2.ename 上层管理者,em1.ename 下属员工 from emp em1 left join emp em2 on em1.mgr=em2.empno order by em1.mgr;

2.6 交叉连接

简单地说就是不需要任何连接条件连接，使用cross join关键字来实现。

--交叉连接dept和emp表，计算出查询结果的行数。
SQL> select count(*) from dept cross join emp;

  COUNT(*)
----------
        56

3. 子查询

3.1 什么是子查询

如果某个操作需要依赖于另一个select语句查询的结果，就需要子查询。

--从emp表中查询部门名称（dname）为RESEARCH的员工信息。
--两个表deptno一样，通过deptno连接两个表，从而定位RESEARCH
SQL> select empno,ename,job from emp where deptno=(select deptno from dept where dname='RESEARCH');

EMPNO ENAME      JOB
----- ---------- ---------
 7369 SMITH      CLERK
 7566 JONES      MANAGER
 7788 SCOTT      ANALYST
 7876 ADAMS      CLERK
 7902 FORD       ANALYST

3.2 单行子查询

返回一行数据的子查询语句。

举例：

--查询出既不是最高工资，也不是最低工资的员工信息。
SQL> select empno,ename,sal from emp where sal>(select min(sal) from emp) and sal<(select max(sal) from emp);

3.3 多行子查询

返回多行数据的子查询语句。当在使用where子句时，必须使用多行比较符（IN、ANY、ALL）

举例（in）：外查询尝试与子查询结果中的任何一个结果进行匹配。

--查询不是SALES部门的员工信息。
SQL> select empno,ename,job from emp where deptno in(select deptno from dept where dname<>'SALES'）;

EMPNO ENAME      JOB
----- ---------- ---------
 7782 CLARK      MANAGER
 7839 KING       PRESIDENT
 7934 MILLER     CLERK
 7566 JONES      MANAGER
 7902 FORD       ANALYST
 7876 ADAMS      CLERK
 7369 SMITH      CLERK
 7788 SCOTT      ANALYST

8 rows selected

举例（any）：必须与单行操作符结合使用，并且返回行只要匹配子查询的任何一个结果即可。

--查询工资大于10号部门的任意一名员工工资的其他部门员工信息。
SQL> select deptno,ename,sal from emp where sal>any(select sal from emp where deptno=10) and deptno<>10;

DEPTNO ENAME            SAL
------ ---------- ---------
    20 SCOTT        3000.00
    20 FORD         3000.00
    20 JONES        2975.00
    30 BLAKE        2850.00
    30 ALLEN        1600.00
    30 TURNER       1500.00

举例（all）：必须与单行运算符结合使用，并且返回行必须匹配所有子查询结果。

--查询工资大于30号部门所有员工工资的员工信息。
SQL> select deptno,ename,sal from emp where sal>all(select sal from emp where deptno=30);

DEPTNO ENAME            SAL
------ ---------- ---------
    20 JONES        2975.00
    20 SCOTT        3000.00

3.4 关联子查询

简单地说，就是前面两个查询的外查询和内查询是分开执行的，互不干扰。但是有些子查询的内查询的执行需要借助于外查询，外查询的执行也离不开内查询，这是就需要使用关联子查询。

举例：

--检索工资大于同职位平均工资的员工信息。
SQL>  select empno,ename,sal,job from emp f where sal>(select avg(sal) from emp where job=f.job) order by job;
