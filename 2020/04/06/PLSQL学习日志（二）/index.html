PLSQL学习日志（二）

PLSQL

过程、函数、包

1.1 存储过程

1.1.1 创建存储过程

存储过程是事先经过编译并存储在数据库中的一段SQL语句的集合，调用存储过程可以简化应用开发人员的很多工作，减少数据在数据库和应用服务器之间的传输，对于提高数据处理的效率是有好处的。

优点：

  • 允许模块化程序设计，就是说只需要创建一次过程，以后在程序中就可以调用该过程任意次。
  • 允许更快执行，如果某操作需要执行大量SQL语句或重复执行，存储过程比SQL语句执行的要快。
  • 减少网络流量，例如一个需要数百行的SQL代码的操作由一条执行语句完成，不需要在网络中发送数百行代码。
  • 更好的安全机制，对于没有权限执行存储过程的用户，也可授权他们执行存储过程。

1.1.2 创建存储过程的语法

create [or replace] procedure 存储过程名（param1 in type，param2 out type）
as
   变量1 类型（值范围）;
   变量2 类型（值范围）;
begin
   语句;
exception
   when others then
        rollback;
end;

1.1.3 存储过程的参数

oracle为了增强存储过程的灵活性，提供向存储过程传入参数的功能。参数是一种向程序单元输入和输出数据的机制。具体参数模式如下：

  • in是参数的默认模式，这种模式就是在程序运行的时候已经具有值，在程序体中值不会改变。参数值由调用方传入，并且只能被存储过程读取。

eg. 创建一个过程，定义三个in模式变量，然后将这三个变量的值插入到dept表中，代码如下：

create or replace procedure insert dept(
    num_deptno in number,    --定义N模式的变量,它存储部门编号
    var_ename in varchar2,    --定义N模式的变量,它存储部门名称
    var_loc in varchar2)is
begin
    insert into dept        
    values(num_deptno,var_ename,var_loc);    --向dept表中插入记录
    commit;        --提交数据库
end insert dept;

注意：存储过程的参数不可以指定长度。

在调用这个存储过程时，需要传入参数来保证执行部分有值参与操作。具体有以下三种传递方法：

  1. 指定名称传递

    语法如下：

    过程名称（参数名称1=>参数值[,参数名称2=>参数值2]）

    拿上一小节的过程举例

    begin
 insert_dept(var_ename=>'采购部';var_loc=>'成都',num_deptno=>15);
end;

可以看到，传递参数值和参数的定义顺序无关，但与参数的个数有关。

  1. 按位置传递

指定名称传递直观易读，但是在参数过多时，会显得代码冗长。所以可以直接按位置传递，这种方式和参数定义顺序有关，必须和过程中定义的参数顺序相同。

begin
 insert_dept(15,'采购部';'成都');
end;
  1. 混合方式传递

即将二者有点结合在一起。

exec insert_dept (15,var_loc=>'成都',var_ename=>'采购部');

  • out 模式定义的参数只能在过程体内部赋值，表示该参数可以将某个值传递回调用他的过程。

  • in out 表示高参数可以向该过程中传递值，也可以将某个值传出去。

  • in参数的默认值 oracle在声明in参数的同时给其初始化一个默认值没。这样在运行过程时，如果没有参数，就会使用这个默认参数。

注意：在使用in模式时，若是使用默认参数可能会出现传值的顺序不固定的情况，所以

​ 最好使用指定名称传递，这样程序就不会出现混乱。

1.1.4 删除存储过程

将过程从内存中删除，释放内存空间，用以下语句：

drop procedure count_num;

eg. 定一个存体过程:获取给定部门的工资私(通过out参数),要:部门号和工总定义方数

--定一个存体过程:获取给定部门的工资私(通过out参数),要:部门号和工总定义方数
create or replace procedure get_sal2 (dept_id number,sumsal out number)
is
       cursor salary_cursor is select salary from employees where department_id=dept_id;
begin
       sumsal :=0;
       for c in salary_cursor loop
           sumsal :=sumsal+c.salary;
       end loop;
       dbms_output.put_line(sumsal);
end;

调用过程并传参

declare
  v_sal number(10):=0;
begin
  get_sal2(80,v_sal);
end;

1.2 函数

1.2.1 创建函数

与存储过程的功能类似，但是不同的是函数有一个返回值，且不需要单独的语句调用。Oracle创建函数是通过PL/SQL自定义编写的，通过关键字function按照自己的需求把复杂的业务逻辑封装进PL/SQL函数中，函数提供一个返回值，返回给使用者。这样使用者就不需要去理解业务逻辑，把PL/SQL函数中的业务逻辑交给专门的开发人员进行编写。

PL/SQL函数主要有下面几部分组成：

  1. 输入部分：PL/SQL函数可以有输入参数，在调用函数时，必须给输入参数赋值。
  2. 逻辑计算部分：逻辑计算部分是由PL/SQL块组成业务逻辑计算部分。这部分主要是通过输入参数、表数据、SQL计算函数等进行逻辑计算得到想要的结果。
  3. 输出部分：通过逻辑计算部分，我们会得到一个函数的唯一返回值进行返回（函数必须要有返回值）。

语法结构如下：

create [or replace] function 函数名 ([p1,p2...pn])
--中括号里面是入参，oracle创建函数可以不需要入参。
return datatype
is|as
--声明部分 
begin
--PL/SQL程序块
end

eg. 创建一个函数，输出hello world，并且给函数一个字符串参数

--返回一个hello world！
create or replace function hello_world(v_logo varchar2)
 return  varchar2
 is
begin
  return 'hello world!'||v_logo;
end;

调用函数

  • 运用PLSQL语句编写
begin
  dbms_output.put_line(hello_world('杨帆')); 
end;
  • 直接使用sql语句
select hello_world('杨帆') from  dual;

1.2.2 删除函数

Oracle函数的修改是通过or replace关键词对原有的函数进行编辑覆盖，因此可以通过pl/sql developer工具直接点击右键进行编辑修改，修改完点击执行，即可完成覆盖。

Oracle函数的删除是通过drop命令进行删除的，语法结构如下：

drop function 函数名;

1.3 程序包

1.3.1 程序包的规范

该“规范”用于规定在程序包中可以使用哪些变量、类型、游标和子程序(指各种命名的 PLSQL块),需要注意的是程序包一定要在“包主体”之前被创建,其语法格式如下：

CREATE OR REPLACE PACKAGE 程序包名称 IS
    规范内声明的变量；
    规范内声明的类型；
    规范内定义的游标；
    规范内声明的函数；--但定义参数和返回值类型，不包括函数体；
    规范内声明的存储过程；--但仅定义参数，不包括存储过程主体；
END；

1.3.2 程序包的主体

程序包的主体包含了在规范中声明的游标,过程和函数的实现代码。另外,也可以在程序包的主体中声明一些内部变量,程序包主体的名称必须与规范的名称相同,这样通过这个相同的名称 Oracle就可以将“规范”和“主体”结合在一起组成程序包,并实现一起进行编译代码,在实现函数或存储过程主体时,可以将每一个函数成存储过程作为一个独立的 PL/SQL块来处理。与创建“规范”不同的是,创建程序包主体使用create package body语句,而不是 create package语句。建程序包主体的代码如下：

CREATE [OR REPLACE] PACKAGE BODY 包名称 IS
 程序包主体的内部变量；
 游标主体;
 从规范中引入的函数头的声明;
 BEGIN
   功能语句;--这是函数主要功能的实现部分。
 EXCEPTION
   异常处理语句；
 END

 从规范中引入的存储过程头部声明；
 BEGIN
    功能语句;--这是存储过程的功能的实现部分
 EXCEPTION
    异常处理语句；
 END
……
END;
