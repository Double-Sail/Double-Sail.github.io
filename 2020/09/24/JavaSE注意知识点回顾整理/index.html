JavaSE注意知识点回顾整理

经理过一段时间的java开发，对于java的基本语法已有了一定的基础。在有一定熟练度和理解的基础上，再次对相关有用的知识进行回顾，总结一些要注意的知识点。

  • 方法的调用

    注意Static关键词的使用，静态方法可以直接进行类名点方法进行调用，而动态方法需要先实例化对象，再用对象名点方法，进行方法的调用。

  • 构造器

    当new一个对象的时候，实际上就是在调用构造器。而构造器的根本作用就是为了初始化值。

    • 直接调用没有赋值的无参构造，会直接返回相应对象的初始值，即String返回null，int返回0等。在无参构造中进行this.属性的赋值操作，可以对属性进行初始化值。

    • 如果定义带参构造，如果没有无参构造的情况下，实例化对象的时候，一定需要提供参数，否则报错，所以在定义有参构造的时候，最好还是添加显示的无参构造。

    • 构造器和类名相同，且无返回类型，void也不可以。

    • 直接定义一个子类对象，那么在构造时，首先会先执行父类的无参构造方法，其后才会进行子类的无参构造方法。

  • 创建对象的内存分析

    注意事项就是在程序运行时，首先会将一些方法体写进方法区。正式开始进行操作，首先将main方法压栈，然后实例化对象操作开始，首先将对象名写进栈，然后new的实例化对象，写进堆内存，这个实例化的对象进行的相关的操作，即是对方法区中的方法体的相关参数的引用。

    Student student = new Student();

    其中，student会进入栈，new student（）实例化对象，写进堆内存。而这个实例化的学生对象进行的名字赋值，以及学习等方法的实行，是对方法区中的Student（）方法进行调用。

    其中，静态方法区会在类进行加载的时候，就直接存入方法区，使得可以直接调用，而不需要在对象生成之后，才可以调用。

  • 方法的重写：需要有继承关系，子类重写父类的方法

    • 方法名必须相同。
    • 参数列表必须相同。
    • 修饰符：范围可以扩大，但不可以缩小。private < protected < default < public;
    • 抛出异常：范围缩小，但不能扩大。
    • 子类方法和父类必须一致；方法体不同！
    • 重写的原因就是，父类的一些方法功能，子类不一定需要，或者不一定满足。

  • 多态

    • 是方法的多态，属性没有多态。

    • 父类和子类，有联系。

    • 存在条件：继承关系，方法需要重写，父类引用指向子类对象。

    • 注意：

      多态情况下的方法调用，具体执行的方法，需要看实例化的具体类型，如果是子类类型则调用子类的该方法，如果是父类，则调用父类的方法。如果父类和子类都有，调用的是子类的方法，因为子类对该方法进行了重写。不可以用父类调用子类独有的方法。

      且static，private，final三个修饰符修饰的方法，无法进行多态，因为方法不可以被重写。

  • static

    该字符表示静态，修饰为静态的方法，在类加载的时候就已经被载入。但是依然有先后顺序。

    //代码1
static{

}
//代码2
public static void dd(){

}

    其中程序会先执行代码1，再执行代码2。

    静态导入包：

    import static java.lang.Math.random;
//此时可以直接使用random（），而不用使用Math.random()
System.out.println(random());

  • 抽象类（abstract）

    抽象类更像是一种框架，一种约束，类似于法律，抽象方法可以只有方法名，但是没有方法体。但是子类继承之后，必须要对父类的抽象方法进行重写，即必须要实现抽象方法。如果子类也是抽象类，则继续抛给孙类实现。

    • 不可以直接实例化，一种抽象的定义，显然是不可以进行实例化的。
    • 抽象类可以有普通方法，抽象方法必须在抽象类中。

    作用：用来提高开发效率，将一些约定俗成的东西进行抽象。但是有序java对于类是单继承，所以抽象类的用途不如接口。

  • 接口（Interface）

    相对于抽象类，进行更高层次的抽象，只有规范，没有方法。进行约束和实现的分离：面向接口编程。

    接口本质是契约，类似于法律，即“如果你是。。。那么你必须。。。”。对接接口，使用implement，可以进行多继承；

  • 内部类

    内部类定义在外部类的里面

    public class Outer {
    private int id;
    private void out(){
        System.out.println("外部方法");
    }
    public class Inner{
        public void in(){
            System.out.println("内部类的方法");
        }
        public void getId(){
            System.out.println(id);
        }
    }
}
    public class Test {
    public static void main(String[] args) {
        Outer outer = new Outer();
        //通过外部类，来实例化内部类
        Outer.Inner inner = outer.new Inner();
        inner.in();
        inner.getId();
    }
}

    这样便可以通过内部类，得到外部类的私有变量。
