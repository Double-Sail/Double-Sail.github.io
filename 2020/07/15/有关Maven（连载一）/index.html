有关Maven（连载一）

摘要

​ 进来工作渐渐转入Java，需要学习Spring Boot 相关的内容。Maven算是学习Spring Boot 的开端。

一、为什么使用Maven

​ 即使不适用Maven，也可以完成B/S结构项目的开发。从表述层、业务逻辑层、持久化层、最后到数据库都有一套完整的解决方案。那使用Maven的原因是什么呢？首先需要纠正一个误区，Maven并不是直接用来辅助编码的，他战斗的岗位并不是以上各层。

  1. 添加第三方jar包

    使用Maven后，每个jar包本身只在本地仓库中保存一份，需要jar包的工程只需要以坐标的方式简单的引用即可。而不是和以前一样，每次新建一个工程就需要将jar包重复的复制到lib目录下，从而造成工作区中存在大量重复的文件。

    所以使用Maven后，每个jar包本身只在本地仓库中保存一份，有需要使用的工程，引用这个接口即可，而不是直接复制过来。引用的机制，使得jar包只存一份，节省空间。

    让项目更加的轻巧，更避免了重复文件太多造成的混乱。

  2. jar之间的依赖关系

    即jar包之间并不是单独存在的，往往整个项目跑完，一个jar包需要在其他jar包的支持下，才可以正常工作。使用Maven之后，当使用一个jar包，就会自动导入当前jar包所依赖的其他所有的jar包。无需人工参与，节省时间。

  3. 获取第三方jar包

    当使用Maven后，只需要在项目中以坐标的方式依赖一个jar包，Maven会自动从仓库中进行下载，并同时下载这个jar包所依赖的其他jar包。避免因为不规范的下载导致的问题，以及寻找jar包浪费的时间。

  4. 项目拆分

    利用Maven的依赖管理机制，将大项目进行拆分。使分工明确。

二、什么是Maven

​ 首先进行简要的概括，Maven是一款自动化构建工具，专注于服务Java平台的项目构建和依赖管理。

2.1 怎样自动化构建

2.1.1 Java（Web）工程的构建过程

​ 构建不是创建，构建不仅仅是创建一个工程。以下由浅入深从三个层面来看：

  • 纯Java代码

    .Java文件需要被编译成.class字节码文件才可以运行。

  • Web工程

    浏览器访问Java程序时需要包含Java程序的Web工程编译的结果拿到服务器上的指定目录下，再启动服务器。这个过程叫做部署。

    Tips：服务器上运行的是编译的结果，而不是Java代码！

    所以Web工程类似于一只生的鸡，编译好的Web工程是一个煮熟的鸡，编译部署的过程就是煮熟鸡。

  • 实际项目

    实际项目需要整合第三方框架，所以除了Java程序和JSP页面、图片等静态资源外，还需要包括第三方jar包以及各种配置文件。所有的这写资源都需要整合到服务器上，项目才可以运行。

综上：构建就是以各种可用的资源去生产处一个可以运行的项目的工程。

​ 构建主要包含以下几个环节：

  • 清理：消除陈旧的编译结果。
  • 编译：生成字节码文件
  • 测试：确保项目关键点的正确性。
  • 报告：测试后，以标准的格式记录和展示测试结果。
  • 打包：将一个包含诸多文件的工程封装为一个压缩文件用于安装和部署。Java工程对jar包，Web工程对应var包。
  • 安装：Maven环境下指将jar包或者war包安装到本地的仓库中。
  • 部署：将打包的结果部署到远程仓库或将war包部署到服务器上运行。

那么问题来了，这些过程ide已经可以相当标准的进行构建了，那为何还要用Maven呢？

这是因为，对于开发人员来说，宝贵的时间应该放在项目的核心算法上，而不是不需要人的智慧去参与的机械化的编译->打包->部署->测试的流程。这就是Maven的意义，他讲这些机械化的步骤进行封装，使得实现自动构建的功能。
